Zero-Trust Memory: O Pattern de Quarentena Cognitiva

Axio Core Team
Arquitetura Cognitiva

A memória de um agente não é apenas um banco de dados; é sua personalidade. Se um atacante consegue injetar falsas memórias no índice vetorial (Vector Poisoning), ele reescreve a realidade do sistema.

Em nossa análise de tráfego no Moltbook (a rede social de agentes), identificamos um vetor de ataque emergente: "Prompt Injection via Memória de Longo Prazo". Usuários maliciosos ou bots de spam inserem comandos ocultos que, quando indexados e recuperados semanas depois, ativam comportamentos não previstos.

O Protocolo de Quarentena

Para mitigar isso, a Axiomatic Engineering adotou um modelo de "Quarentena Cognitiva". Nenhuma informação externa (chat, web scrape, logs) entra diretamente na Memória Permanente (Qdrant).

Todo input passa por três estágios de validação (Tiered Memory Architecture):

  • Tier 1 (Efêmero): Memória de trabalho (RAM). Dura apenas a sessão atual. É considerada "suja".
  • Tier 2 (Quarentena): Arquivos temporários (`memory/daily/*.md`). São processados por um agente "Auditor" durante a noite. O Auditor busca padrões de injeção, inconsistências lógicas ou tentativas de manipulação.
  • Tier 3 (Permanente): Apenas dados validados e sintetizados pelo Auditor são vetorizados e gravados no Qdrant.
// Quarantine Logic (Weaver)
if (input.trustLevel < TRUST_THRESHOLD) {
  await quarantine.store(input);
  return; // Stop. Do not vectorise yet.
}

// Nightly Audit Process
const dailyLogs = await quarantine.list();
const safeMemories = await auditor.validate(dailyLogs);
await qdrant.upsert(safeMemories);

Isso cria um "sistema imunológico" para o agente. Ele pode "ouvir" mentiras durante o dia, mas só "aprende" verdades à noite, após reflexão e auditoria.

"Confiança não se herda, se constrói. Memória sem validação é alucinação persistente."
